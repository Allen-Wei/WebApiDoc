客户端请求

  1. Http Request Header的格式
    Accept: application/json

  2. 如果请求Body(比如Post/Put请求)中有JSON数据，Content-Type的值为application/json.
  3. 如果请求需要认证的接口要携带认证Cookie.
服务器端接口设计

下面以资源Order为例设计API

  1. REST接口 

    IEnumerable<Order> Get()                                                      //获取所有订单列表
IEnumerable<Order> Get(int page, int count)                                   //获取指定页的订单列表
IEnumerable<Order> Get(string filter, string order, int page, int count)      //获取指定页的订单列表根据条件和排序
Order Get(int id)                                                                   //根据Order Id获取Order

Put(Order order)                                                                    //更新订单
Post(Order order)                                                                   //新增订单
Post(Order[] orders)                                                                //新增多条订单
Delete(int id)                                                                      //删除指定订单
Delete(string filter)                                                               //删除多个订单根据条件

  2. RSP接口

    [HttpGet]
UpdateStatus(int id, strign newStatus)                                              //更新订单状态
[HttpPost]
Pay(Order order)                                                                    //支付订单
  3. REST接口可以不标注Http Verbs(Get/Post/Put/Delete), 但是Action命名必须以Verb开始, 且需要添加 /// <style>REST<style> 注释.
  4. RSP必须添加Http Verb注解. 当使用RSP接口时考虑这是不是新的资源，如果是最好新建一个资源采用REST来设计API.
